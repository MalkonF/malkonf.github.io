Instalando e configurando o SSH(Secure Shell) Authentication Protocol no Debian 10 "Buster"

12 de junho, 2020 por Malkon Faria

O que é o Protocolo de Autenticação de Shell Seguro?

É um protocolo que garante a um usuário fazer login em uma máquina remota de forma segura, utilizando vários métodos de autenticação(chave pública, senha ou baseado em host) e além disso ele também proporciona a utilização de vários serviços de rede de maneira segura sobre uma rede insegura através de um canal de comunicação encriptado. Uma rede insegura, nesse contexto, se diz respeito a uma rede onde os dados não são criptografados e trafegam em texto plano. Uma rede que usa o protocolo TCP/IP é um exemplo de rede insegura, já que por padrão os dados que trafegam por estas redes não são criptografados. Para verificar isso, experimente usar o protocolo TELNET(que roda sobre TCP/IP) e interceptar os dados com um sniffer, com certeza você visualizaria todos os dados sem nenhum problema.

Como instalar?

No servidor onde você quer se conectar, faça:


          sudo apt-get update
          sudo apt-get install openssh-server

Para verificar a versão que foi instalada entre com:


          ssh -V
Verifique se o serviço está ativo: 

          service ssh status

Ele deve estar com status "running", isso significa que está pronto para aceitar conexões

No computador que você vai usar para se conectar ao servidor SSH, faça:


          sudo apt-get update
          sudo apt-get install openssh-client

Como faço para me conectar?

Bem, para se conectar é necessário informar o endereço IP do servidor SSH juntamente com um nome de usuário que exista naquele servidor. Partindo do pressuposto que não exista nenhum firewall ativo no servidor SSH(nos próximos capítulos deste artigo vamos ensinar como configurar um firewall para liberar acesso ao SSH) você pode se conectar entrando com:


          ssh -l usuário endereço_ip_servidor_shh

por exemplo:


              ssh root 167.99.203.217

ou:


                ssh root@167.99.203.217
